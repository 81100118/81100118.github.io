1 基础

  • 算法记得写Input和Output
  • algorithm和heuristic的区别：heuristic是一个idea，guide，不保证一定有好的解法
  • BST: 左子树所有节点<根<右，remove:若有1个子，则由父接管子；若有2个子，则和子树的最大节点（必定是叶子）交换，再删除被交换的叶子

算法： - MST: 贪心：kruskal，how to detect cycles? 提到union-find structure. 每堆连在一起的节点形成一棵树。一开始每个节点都是一棵树，父是自己。 If the edge does not cause a cycle（根不相同的话，就not cause a cycle）, we add the edge and make union on the two different components connected by the edge. detect cycle: O(log|V|). kruskal总复杂度：|E|log|E|+|E|log|V| = |E|log|V| - MST: Prim: D为某节点和所有相邻节点的边的最短边长。选一个起始节点s，令D(s) = 0, 其他节点D = inf，循环：从未连接的节点中选择D最小的，更新所有未连接节点的D及其predecessor；选择最小D的节点，连接它和它的predecessor，循环结束 - counting inversions: inversion def: a[0..n-1]中，如果a[i]>a[j]，i<j，那么(a[i], a[j])这个pair是一个inversion. 算法：divide and conquer, nr_inv(total) = nr_inv(L) + nr_inv(R) + nr_inv(LR merge时产生的inv)。 merge时，进行排序，与此同时计数 merge产生的inversion。

2 复杂度理论

  • \(f = O(g)\) (f小于等于g), \(f = \Omega(g)\)（f大于等于g）, \(\Theta\)（等于）
  • loglogn < logn / loglogn
  • \(\alpha(n)\): inverse ackerman's func, 增长巨慢，只会略大于1
  • stable marrage problem: 存在（m,w）使得m比起他老婆更喜欢w，w比起她老公更喜欢m。这就是不稳定的。Gale-Shapley算法：男人依次根据志愿求婚；女人发现求婚者比现任更好的话就换男人，否则不换。如此循环直到没有free man。复杂度：O(n^2)因为每个男人最多把所有女人都求婚一遍。
  • log*: log*x = n, 等价于 2(2(2^...)) n-1个2 < log* x <= 2(2(2^...)) n个2
  • union-find: 1次find: O(logn) m次find: O((m+n)log*n) 但不是最接近的。Inv Ackerman才是最接近的
  • Ackerman's func: 太复杂了。。不想看
  • Master Theorem: \(T(n) = aT(n/b) + f(n)\) （记住顺序：a, b, f(n)）； 若\(f(n\))比\(n^{\log_b a}\)小，则\(T(n) = n^{\log_b a}\)（记住\(n^{\log_b a}\)。[怎么记：正好是上一行的顺序反一下] \(f(n)\)和他谁大，总的复杂度就是谁，可以理解为瓶颈效应） ​= ，则\(T(n) = n^{\log_b a} \log n\) （记住a=b时，复杂度是nlogn） ​ > ，则\(T(n) = f(n)\) （和(1)类似，瓶颈效应）
  • computational problem: 计算机能解决的一个或一系列问题
    • decision problem: answer is boolean
    • search problem: answer is string （似乎是最难的问题）
    • counting problem: answer is #solution of a search problem
    • optimization problem: answer is the most blabla solution of a search problem
    • function problem: only 1 answer
  • turing machine: \[M: (\Sigma \cup \{b\}) \times Q \longrightarrow (\Sigma \cup \{b\}) \times \{-1, 0, 1\} \times Q\] （根据当前字符及状态，改变字符、磁头移动、改变状态）

  • NPC: 既是NP，又是NP-hard
  • 证明3-SAT是NPC
    • 3-SAT是NP，因为SAT是NP，3-SAT只是SAT的一种特殊情况
    • SAT可以转化为3-SAT，说明3-SAT >= SAT
  • 所以，如何证明某个新问题是NPC:
    • 证明新问题是NP
    • 找到一个已知NPC问题，证明它可以转化为这个新问题。因为已知NPC是NP-hard，新问题不比它简单，所以新问题也是NP-hard
    • 所以新问题是NPC

3 dp

  • DJ算法：no negative edge。和prim差不多，不过D为从起点到那个点的最短距离，而不是相邻点到那个点的最短距离。算法概括：初始化起点d为0，（访问d最小的点，更新未访问neighbor）如此循环
  • BF算法：一个node的dist初始化为0，其余为inf。依次求从起点通过至多1条边、2条边、3条边...可以达到各个顶点的最短距离。要求：no neg cycle. How to detect neg cycle: 1 more iteration on BF and see whether there is a node whose shortest path length changes. if so, there is neg cycle.

4 网络流

  • BFS: O(|E|)
  • Ford-Fulkerson算法：不断寻找增广路，复杂度不确定
  • Edmonds-Karp算法：FF算法改进，用bfs来寻找增广路，复杂度：O(|V||E|^2)
